## What I learned

### GitHub & Pages
- The `username.github.io` repo is special: it's a **user site** and auto-deploys to `https://username.github.io/` from the `main` branch. Other repos deploy as project sites to `https://username.github.io/repo-name/`.
- The deploy itself is fast (usually under a minute) but the browser cache isn't. A "stale" page after pushing is almost always a cache issue, not a deploy issue.
- Fix: hard refresh (`Cmd+Shift+R` / `Ctrl+Shift+R`) or open in incognito to confirm what visitors actually see.

### Prompting AI for code
The prompt structure the class taught actually matters. It's not just "make me a website" — it's:
- **Description** (who, what for)
- **Requirements** (constraints: single file, no frameworks, what sections to include)

The constraints are what keep the AI from producing bloated output. When I gave it a tight spec, the result was clean and editable. Vague prompts get generic results.

### The "Big Shift"
The point isn't to memorize HTML tags — it's to understand structure well enough to read, evaluate, and debug what AI generates. Even on day 1 I can spot obvious issues (broken nesting, inline styles everywhere, missing semantic tags), which is the whole skill the class is teaching.

### Markdown
- Headings (`#`, `##`), bullets (`-`), inline code with backticks, fenced code blocks, and links `[text](url)`
- GitHub renders `.md` files automatically — what you see in the repo *is* the rendered version
