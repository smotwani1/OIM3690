CSS Layout (Flexbox & Grid)
- Flexbox is for one-dimensional layout. My nav bar uses it, and `gap` spaces the items without margin hacks
- CSS Grid is for two-dimensional layout. My flashcards use `auto-fill` + `minmax()` so cards reflow on their own, responsive without a media query:
- `box-sizing: border-box` makes padding count inside the width, so my cards don't overflow.

Responsive design
The `<meta name="viewport">` tag is what makes mobile scaling work at all.
- Media queries collapse my layouts on phones: `@media (max-width: 520px)` switches the "At a glance" grid and the checklist to a single column.
- Fluid type with `clamp()` scales headings smoothly instead of jumping: `font-size: clamp(36px, 7.5vw, 58px)`.

Forms
- Every input needs a `<label>`; input types (`email`, `text`, `select`, `textarea`) and validation attributes (`required`, `minlength`, `type="email"`) do a lot of work before any JS.
- Class covered submitting to a server, my form is client-side only, so I learned what server submission would require (a backend/service like Formspree) and why a static GitHub Pages site can't do it alone.

JavaScript (DOM + events)
- `document.getElementById` / `querySelector` to grab an element, `addEventListener` to react to a `click` or `submit`, and then modify text/attributes:
- I learned to flip the theme via a `data-theme` attribute + CSS variables instead of setting styles directly in JS, keeps styling in CSS.

## AI interactions

<!--
One block per interaction. Add more as needed.

### Interaction 1
- **What I asked**: "Split my single-page guide into a multi-page site that meets the assignment requirements, 3+ pages, semantic HTML, an external stylesheet, Grid/Flexbox, responsive, favicon."
- **What AI gave me**: Five linked HTML pages with a shared `<header>`/`<nav>`, an external `styles.css`, Flexbox nav, Grid layouts, media queries, a dark-mode toggle, and a favicon.
- **What I kept / changed / rejected**: Kept the structure. Before accepting it I read through each page and marked the CSS I didn't recognize (`clamp()`, `auto-fill`, `box-sizing`), asked AI to explain just those, then confirmed they matched the slides. 

### Interaction 2
- **What I asked**: "Explain the difference between using Flexbox and Grid, when do I reach for each?"
- **What AI gave me**: Flexbox is one-dimensional (a row or a column, like a nav bar); Grid is two-dimensional (rows and columns at once, like a card layout). `auto-fill` + `minmax()` lets a grid be responsive without media queries.
- **What I kept / changed / rejected**: Kept it as a rule of thumb and used it to justify why my nav is Flexbox and my flashcards are Grid.

### Interaction 3 (the VERIFY step that mattered most)
- **What I asked**: I had an AI design tool generate a mockup of my "At a glance" section.
- **What AI gave me**: A good-looking layout, but with invented facts: a wrong address (66 Mt Auburn St instead of the real 550 Arsenal St), "log book, valid ID" instead of the actual requirements, and it missed the parking-brake rule entirely.
- **What I kept / changed / rejected**: Rejected all the AI-generated content. Kept only my verified facts from the official RMV docs and Driver's Manual. This was the clearest lesson of the week: AI generates confident, wrong information, and verifying against the source is the whole point of my product.
